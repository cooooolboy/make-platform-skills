# canvas-table 单元格编辑默认规范

## 背景

新 Make POC 中出现了单元格编辑交互偏差：弹窗型字段进入编辑后没有立即打开弹窗，内联编辑器没有铺满活动单元格，部分字段进入编辑后没有正确回显，且存在未修改也触发保存的风险。

## 本次调整

- 在 `canvas-table-integration` 的 Track B 中补充 ExpensePoc 验证过的单元格编辑默认规则。
- 新增 `references/make-cell-edit-defaults.md`，集中说明弹窗型字段、内联字段、值回显、提交值、未变更跳过保存、受控回填和回滚规则。
- 更新编辑契约、生命周期、字段编辑映射、Track B workflow 和常见问题，明确：
  - 日期、日期范围、下拉、人员、部门、附件等弹窗型编辑器必须在同一次编辑激活中打开。
  - 文本、文本域、数字、金额、百分比等内联编辑器必须铺满活动单元格，不得出现额外边框、外边距或外层 padding。
  - 编辑器初始化使用原始后端值，不能使用格式化展示文本。
  - 初始化值回退只按 `undefined` 判断，避免把合法的 `null`、空字符串或空数组误认为缺失值。
  - 单行文本与多行文本的 Enter 行为分开描述，避免 textarea 被误生成成回车即提交。
  - `renderValue`、`submitValue`、`displayValue` 要分离。
  - 未修改的值关闭编辑器时不调用保存接口，也不回填表格数据。
