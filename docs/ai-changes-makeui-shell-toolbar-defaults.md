# makeui Shell 与列表工具栏默认规则

## 2026-05-25

- 将 `skills/makeui` 修订标识更新为 `0.3.23`。
- 将 ExpensePoc 风格的密集对象管理页作为默认布局基线：左侧导航、扁平工作区头部、局部工具栏直接接表格、无额外列表标题卡片。
- 明确侧边栏布局是默认要求，但背景颜色不默认指定为深色，应跟随项目主题、设计系统 token 或用户品牌要求。
- 明确默认左侧栏包含品牌区、分组标签、单行对象项和清晰选中态；品牌区可以展示应用级元信息，对象导航项不展示副标题。
- 明确左侧导航对象项默认只展示主名称，不展示对象描述、副标题、辅助说明或第二行文案。
- 明确右侧工作区头部默认只展示当前对象或模块标题，不展示副标题、schema 摘要、来源说明或概览文案。
- 明确列表刷新按钮必须放在表格上方的页面局部工具栏中，靠近搜索或次要操作区域。
- 明确刷新按钮不得放在全局头部、对象标题头部、表格表头行、canvas-table header 区或字段列头区域。
- 明确默认对象列表不插入重复标题、记录数、schema 来源说明的白色摘要卡片。
