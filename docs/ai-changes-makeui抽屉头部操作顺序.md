# makeui 抽屉头部操作顺序

## 变更内容

- makeui 版本更新到 `0.3.10`。
- 明确新建、编辑、详情 Drawer 的头部默认顺序：
  - 左侧标题区域：全屏/退出全屏图标、模式或状态标签、标题。
  - 右侧操作区域：业务操作按钮、保存/取消等上下文操作，最后一个元素必须是关闭图标。
- 明确关闭按钮默认是 icon-only，使用 `关闭` 作为 accessible name/title，不默认渲染可见的 `关闭` 文字按钮。
- 禁止在左侧标题区域放关闭 X。
- 禁止重复关闭入口，例如左侧一个 X，右侧再一个 `X 关闭` 按钮。

## 行为说明

- 全屏按钮属于标题上下文，所以默认位于头部左侧。
- 关闭抽屉是退出当前浮层的全局动作，默认固定在头部最右侧。
- 保存、取消、编辑、删除、推进状态等业务动作都应位于关闭按钮左侧。

## 验证

- 使用 `rg` 检查源目录和安装目录中的 `0.3.10`、`icon-only close`、`left title area`、`duplicate close` 等关键规则。
- 同步源目录到 `~/.agents/skills/skills-makeui`。
